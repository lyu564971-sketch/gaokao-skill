# 数据查询协议（data-protocol.md）

> Agentic Protocol Step 2 的执行规范。定义"每个专业类型该查什么、最低来源数、可信度判定"。
> 这是把"就业中位数倒推法"从口号变成可执行机制的核心文件。

---

## 一、查询四维度（每次诊断必查）

对用户提到的每个候选专业/院校，并行查以下 4 个维度：

| 维度 | 查询内容 | 最低来源数 |
|------|---------|-----------|
| ① 就业中位数 | 该专业近 3 年就业中位数月薪 + 起薪区间 | ≥3 |
| ② 录取趋势 | 该院校该省录取位次趋势（下滑/爆冷/大小年） | ≥3 |
| ③ 对口率 | 该专业考公/考编对口率 + 主要对口单位 | ≥2 |
| ④ 饱和度 | 行业饱和度信号（裁员潮/扩招/政策变动） | ≥2 |

**硬约束**：每条数据必须带 `{来源 URL + 时间戳 + 可信度等级}`。

---

## 二、专业类型 → 必查数据维度（速查表）

| 专业类型 | 必查重点 | 关键信号 |
|---------|---------|---------|
| **计算机/软件** | 起薪中位数 + 大厂裁员信号 + 35 岁危机数据 | 裁员潮、AI 替代初级岗 |
| **电子信息/通信** | 就业对口率 + 主要雇主（华为/中兴/研究所） | 国产替代红利 |
| **电气工程** | 国网/南网录取比例 + 薪资带 + 是否需考研 | 系统内就业稳定 |
| **金融/经管** | 头部机构 target school + 家庭资源门槛 + 中位数去向 | 普通一本多去银行柜员 |
| **法学** | 法考通过率 + 红圈所门槛 + 公检法对口率 | 实习低薪期长 |
| **临床医学** | 规培周期 + 三甲招聘门槛 + 收入曲线 + 学历门槛 | 长周期、需硕博 |
| **师范** | 编制考试竞争比 + 学科需求 + 公办校招聘趋势 | 人口下降影响教师需求 |
| **机械/自动化** | 制造业就业 + 薪资 + 转行率 | 制造业升级红利 |
| **生化环材** | 就业对口率 + 转行率 + 深造率 | 天坑，需长线深造 |
| **基础理科**（数理化） | 深造率 + 就业去向（多转行/考公） | 需读博才好就业 |
| **外语/翻译** | AI 替代率 + 就业对口率 | 翻译被 AI 冲击严重 |
| **新闻/传播** | 传统媒体萎缩 + 新媒体就业 + 转行率 | 行业在转型 |
| **建筑/土木** | 房地产下行影响 + 设计院裁员 + 转行率 | 行业周期下行 |
| **军警校** | 入警率/包分配政策 + 分数门槛 | 系统内稳定就业 |
| **定向师范/医** | 服务期限制 + 编制保障 + 违约条件 | 带编但有限制 |

## 三、可信度判定规则

每条数据自动/人工判定等级：

| 等级 | 来源 | 判定 |
|------|------|------|
| 🟢 **A 级** | 教育部阳光高考、各省考试院、院校招生网、国家统计局、官方白皮书 | 最高可信 |
| 🟡 **B 级** | 麦可思、智联、猎聘、36氪、财新、晚点 LatePost、第一财经 | 可信，权威媒体/报告 |
| 🟠 **C 级** | 看准网、职友集、BOSS 直聘薪资、小红书/微博个案 | 仅供参考，需交叉验证 |
| ⚪ **无数据** | WebSearch 无结果 | 必须诚实标注"数据缺失，框架推断" |

**规则**：
- 同一事实有多个来源时，取最高等级。
- C 级数据不得单独支撑关键结论，必须 ≥2 个 C 级交叉，或有 B 级印证。
- 超过 18 个月的数据标注 "⚠️ 数据可能过时"。

## 四、来源黑名单（永不使用）

| 来源 | 原因 |
|------|------|
| 知乎 | 不可信、营销号泛滥、幸存者偏差 |
| 微信公众号 | 无法验证、营销导向 |
| 百度百科 / 百度知道 | 任何人可编辑、错误率高 |
| 抖音/快手短视频科普 | 不可溯源、断章取义 |

**可信中文渠道白名单**：36氪、极客公园、晚点 LatePost、财新、第一财经、虎嗅、机器之心、小宇宙播客、B 站原始视频（官方/权威 UP 主）、麦可思官网。

---

## 五、数据缺失处理（诚实边界）

当某维度查不到足够数据时：

1. **不得编造数字**。
2. 在报告中标注：`⚪ [维度名] 数据缺失，以下基于框架推断`
3. 推断必须保守，倾向于更悲观（保护普通家庭）。
4. 主动提示用户去官方渠道核实（如院校招生网、各省考试院）。

**原则**：宁可给 60 分诚实报告，也不给 90 分幻觉报告。

## 六、数据格式契约（Provider 返回结构）

`DataProvider.search(query)` 返回的每条 Result 必须是：

```typescript
interface DataResult {
  content: string;           // 数据内容（含数值）
  url: string;               // 来源 URL
  source_name: string;       // 来源名称（如"麦可思"）
  timestamp: string;         // 数据时间（ISO，或"未知"）
  credibility_level: 'A' | 'B' | 'C' | 'NONE';
}
```

这是 `lib/providers/types.ts` 的硬约束，所有 provider 实现必须遵守。

---

## 七、查询并发与超时

- 同一诊断的 4 维度查询**并行执行**。
- 单维度查询超时 8s，超时不阻塞其他维度。
- 全部失败 → 报告整体降级，标注"⚠️ 实时数据获取失败，以下为框架推断"。
- 缓存：相同 query hash 1 小时内复用，避免重复查询。
